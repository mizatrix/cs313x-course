# 1. Vector Space Model (VSM) — Introduction

## Concept

The transition from **Boolean Retrieval** (exact match, no ranking) to **Ranked Retrieval** using the Vector Space Model. Documents and queries are represented as high-dimensional vectors in term-space.

## Key Ideas

- Boolean model: feast or famine problem (too many or too few results)
- VSM allows **ranking** results by relevance
- Each term in the vocabulary is a dimension
- Documents live in a shared vector space

## 🎨 Visualization Ideas

| Visual | Description |
|--------|-------------|
| **Boolean vs. VSM comparison** | Side-by-side showing Boolean's exact match vs. VSM's ranked results |
| **2D/3D Vector Space Plot** | Documents as colored dots in a term-space with axes labeled as terms |
| **Interactive exploration** | Click a query to see where it lands in the vector space relative to documents |
