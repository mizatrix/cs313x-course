Soundex Algorithm

🔊 Soundex Algorithm

Watch how names are encoded into phonetic codes step by step. Names that sound alike produce the same code — useful for matching "Smith" and "Smyth"!

Encoding Pipeline

1
Keep First Letter
Retain the first letter of the name (uppercased).
2
Map Letters → Digits
Replace consonants with digits using the Soundex table. Drop A, E, I, O, U, H, W, Y.
3
Remove Adjacent Duplicates
If two adjacent digits are the same, keep only the first.
4
Remove Zeros
Drop all 0s (vowels and H/W/Y that were mapped to 0).
5
Pad / Truncate to 4 Characters
Ensure the code is exactly 1 letter + 3 digits. Pad with 0s or truncate.

What is Soundex?

Soundex is a phonetic algorithm that encodes names by their pronunciation, so that names that sound similar get the same code — even if they're spelled differently.

  • Used in databases, genealogy searches, and spell-checkers
  • Originally patented in 1918 by Robert C. Russell
  • Produces a 4-character code: first letter + 3 digits

📜 Algorithm Pseudocode

SOUNDEX(name)
 1  name ← UPPERCASE(name)
 2  code ← name[0]                   // keep first letter
 3  prevDigit ← SOUNDEX-DIGIT(name[0])
 4
 5  for i ← 1 to LENGTH(name) do
 6    digit ← SOUNDEX-DIGIT(name[i])
 7    if digit ∈ {1,2,3,4,5,6} and digit ≠ prevDigit then
 8      code ← code + digit         // append digit to code
 9      prevDigit ← digit
10    else if digit = 0 and name[i] ∉ {H, W} then
11      prevDigit ← 0               // reset for vowel separators
12    if LENGTH(code) = 4 then break
13
14  while LENGTH(code) < 4 do
15    code ← code + "0"             // pad with zeros
16  return code[0..3]

Mapping Table:
  B,F,P,V → 1    C,G,J,K,Q,S,X,Z → 2
  D,T → 3        L → 4
  M,N → 5        R → 6
  A,E,I,O,U,H,W,Y → 0 (drop)
