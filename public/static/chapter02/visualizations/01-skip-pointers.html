What are Skip Pointers?

In an inverted index, each term has a posting list — a sorted list of document IDs where the term appears. To find documents containing both terms, we intersect their posting lists.

  • Without skip pointers: we compare element-by-element → O(n+m)
  • With skip pointers: we place "express lanes" every √n elements. When the current value in one list is much smaller, we check if the skip target is still ≤ the other list's current value. If yes, we jump — saving many comparisons.

The skip interval is typically ⌊√len⌋. This balances the trade-off between too many skips (wasted pointer space) and too few (no speedup).

📜 Algorithm Pseudocode

INTERSECT-WITH-SKIPS(p₁, p₂)
 1  answer ← ⟨ ⟩
 2  while p₁ ≠ NIL and p₂ ≠ NIL do
 3    if docID(p₁) = docID(p₂) then
 4      ADD(answer, docID(p₁))
 5      p₁ ← next(p₁)
 6      p₂ ← next(p₂)
 7    else if docID(p₁) < docID(p₂) then
 8      if hasSkip(p₁) and docID(skip(p₁)) ≤ docID(p₂) then
 9        while hasSkip(p₁) and docID(skip(p₁)) ≤ docID(p₂) do
10          p₁ ← skip(p₁)       // take the skip
11      else
12        p₁ ← next(p₁)         // advance linearly
13    else
14      // symmetric: try skip on p₂
15      if hasSkip(p₂) and docID(skip(p₂)) ≤ docID(p₁) then
16        while hasSkip(p₂) and docID(skip(p₂)) ≤ docID(p₁) do
17          p₂ ← skip(p₂)
18      else
19        p₂ ← next(p₂)
20  return answer

Skip interval: ⌊√n⌋ where n = length of posting list
