Why is Tokenization Hard?

Tokenization seems simple — just split on spaces, right? But natural language is messy:

  • A period can end a sentence or be part of an abbreviation (Mr., U.S.A.)
  • An apostrophe can be possessive (John's), a contraction (don't), or part of a name (O'Brien)
  • Hyphens can join compound words (state-of-the-art) or be dashes
  • Numbers may contain periods (3.14), commas (1,000), or slashes (01/15)

Different strategies suit different tasks. A search engine might keep "New York" as one token, while a POS tagger splits it.

📜 Algorithm Pseudocode

SMART-TOKENIZE(text)
 1  tokens ← ⟨ ⟩
 2  consumed[1..n] ← FALSE            // track which chars are used
 3
 4  // Priority-ordered pattern list:
 5  patterns ← [
 6    URL       : /https?:\/\/[^\s]+/   // highest priority
 7    Email     : /\w+@\w+\.\w+/
 8    Abbrev    : /Mr\.|Dr\.|U\.S\.A\./
 9    Date      : /\d{1,2}[/-]\d{1,2}[/-]\d{2,4}/
10    Number    : /\$?\d[\d,]*\.?\d*%?/
11    Compound  : /\w+(-\w+){2,}/
12    Hyphen    : /\w+-\w+/
13    Contraction: /\w+n't|\w+'[msdrv]e?/
14    Word      : /[A-Za-z]+/
15    Punct     : /[^\s\w]/             // lowest priority
16  ]
17
18  for each pattern p in patterns do
19    for each match m of p in text do
20      if no character in m is consumed then
21        ADD(tokens, m)
22        mark m's characters as consumed
23
24  SORT tokens by position
25  return tokens

Key: Higher-priority patterns are matched first.
If a character is already consumed, later patterns skip it.
This prevents URLs from being split into words + punctuation.
