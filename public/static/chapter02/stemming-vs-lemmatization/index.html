Word Porter Stem Lemma Notes

Key Differences

  • Stemming chops off word endings using rules (e.g. remove "-ing", "-ed", "-tion"). Fast but crude — may produce non-words.
  • Lemmatization uses a dictionary + morphological analysis to find the actual root word (lemma). More accurate but slower.

False Positive (FP): Stemmer maps unrelated words to the same root (e.g. "university" & "universe" → "univers").

False Negative (FN): Stemmer fails to map related words to the same root (e.g. "ran" stays "ran" instead of "run").

📜 Algorithm Pseudocode

── STEMMER (Rule-Based) ──
PORTER-STEM(word)
 1  for each step s ∈ {1a, 1b, 1c, 2, 3, 4, 5} do
 2    for each rule (suffix → replacement) in s do
 3      if word ends with suffix then
 4        stem ← word - suffix
 5        if CONDITION(stem) then    // e.g., m > 0
 6          word ← stem + replacement
 7          break                  // only first match per step
 8  return word

── LEMMATIZER (Dictionary-Based) ──
LEMMATIZE(word)
 1  pos ← POS-TAG(word)              // Part-of-Speech tagging
 2  if word ∈ IRREGULAR-TABLE[pos] then
 3    return IRREGULAR-TABLE[pos][word]  // ran→run, better→good
 4  suffix ← DETECT-INFLECTION(word, pos)
 5  if suffix found then
 6    base ← REMOVE-INFLECTION(word, suffix)
 7    if base ∈ DICTIONARY then
 8      return base               // valid word confirmed
 9  return word                     // already in base form
