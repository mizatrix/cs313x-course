How Edit Distance Works

The Levenshtein distance between two strings is the minimum number of single-character edits needed to transform one into the other:

  • Insertion — add a character
  • Deletion — remove a character
  • Substitution — replace a character

The algorithm fills a (m+1) × (n+1) matrix where cell D[i][j] holds the edit distance between the first i characters of the source and the first j characters of the target.

Each cell is computed as: min(D[i-1][j]+1, D[i][j-1]+1, D[i-1][j-1] + cost) where cost = 0 if characters match, 1 otherwise.

📜 Algorithm Pseudocode

EDIT-DISTANCE(source s[1..m], target t[1..n])
 1  for i ← 0 to m do
 2    D[i, 0] ← i                   // base: delete all from source
 3  for j ← 0 to n do
 4    D[0, j] ← j                   // base: insert all from target
 5
 6  for i ← 1 to m do
 7    for j ← 1 to n do
 8      if s[i] = t[j] then
 9        cost ← 0                   // characters match
10      else
11        cost ← 1                   // substitution needed
12      D[i, j] ← min(
13        D[i-1, j] + 1,             // deletion
14        D[i, j-1] + 1,             // insertion
15        D[i-1, j-1] + cost         // substitution or match
16      )
17
18  return D[m, n]
