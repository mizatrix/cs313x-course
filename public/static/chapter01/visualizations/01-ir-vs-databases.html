Visualization 01

IR vs Database Systems

See how the same query is handled differently by a structured database (SQL) versus an Information Retrieval system.

🗄️ Database (SQL)
SELECT * FROM docs
WHERE body LIKE '%smartphone%';
Properties
📋 Data Format: Structured (tables, columns, types)
🔍 Query Type: Exact & Formal (SQL syntax)
📊 Results: Unordered set of exact matches
✅ Correctness: Deterministic (right or wrong)
⚡ Complexity: O(N × L) — sequential scan
⏱️ Speed: Slow (seconds for large datasets)
Results
🔎 Information Retrieval
Query: "best smartphone 2024"
→ Tokenize → Match Index → Rank by TF-IDF
Properties
📋 Data Format: Unstructured (free text, any length)
🔍 Query Type: Natural language, vague intent
📊 Results: Ranked list (best → worst)
✅ Correctness: Probabilistic (relevant or not)
⚡ Complexity: O(1)* — index lookup
⏱️ Speed: Fast (milliseconds)
